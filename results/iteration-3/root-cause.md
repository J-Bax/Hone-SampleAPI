Add .AsNoTracking() to all read-only GET queries in ProductsController to eliminate EF Core change-tracker overhead. The change tracker allocates identity-resolution dictionaries and snapshot copies for every materialized entity, which is pure waste on read-only endpoints. This is the lowest-risk, single-line-per-query change that reduces memory pressure and CPU time, directly lowering p95 latency and raising RPS. Both previous regressions (eager loading and DTO projection) altered query shape; .AsNoTracking() preserves the exact same SQL and result shape while removing only the in-memory tracking cost.
